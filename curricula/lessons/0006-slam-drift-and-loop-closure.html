Loop closure corrects the map—not the last camera frame

2. A SLAM system has two different time horizons

Trackimages + IMU → current motion
Associatematch observations to features
Optimizeadjust poses and landmarks
Recognizerelocalize or close a loop

Front end: what corresponds?

Tracking and data association turn sensor measurements into motion estimates and constraints. They must keep up with incoming data.

Back end: what state best fits?

Optimization combines constraints across keyframes. A newly recognized old place can revise many earlier pose estimates.

ORB-SLAM3 makes the distinction tangible: tracking supplies real-time pose estimates, while local mapping, place recognition, map merging, and loop closing reuse keyframes across much longer spans of time.

3. Why incremental odometry drifts

Each short motion estimate has some error. The next estimate starts from the already imperfect result, so small translation, rotation, timing, calibration, and measurement errors accumulate. Odometry is locally useful but has no guarantee that returning to the starting image means returning to exactly the starting estimated pose.

many small relative estimates → accumulated pose error
revisiting a known place → one new long-range constraint

A loop closure is not “the algorithm noticing drift.” Place recognition proposes that a current observation matches an earlier place; geometric checks establish a constraint between the two poses; optimization then finds a trajectory and map that better satisfy all constraints.

5. Protect the controller from the global correction

Suppose loop closure moves the best global vehicle pose by one metre. Sending that jump directly as the controller's local position can look like an impossible instantaneous motion and create a large control error.

T_map_body = T_map_odom · T_odom_body

T_odom_body stays smooth for local control
T_map_odom absorbs the global correction

This is the purpose of separating the frames: odom is continuous and locally accurate but allowed to drift; map is globally corrected but allowed to jump. The exact integration contract still matters—PX4 does not automatically infer ROS frame semantics from a pose stream.

6. Relocalization is related, but not identical

Loop closure

Tracking continues, and the system recognizes that the current place was seen earlier. The new constraint corrects accumulated inconsistency.

Relocalization

Tracking was lost or restarted. The system recognizes a known place to recover its pose relative to an existing map.
