PX4 listener commands

A copy-ready reference for tracing the vehicle attitude, commanded gimbal attitude, and resulting gimbal attitude in Lesson 0001.

The three-command trace

1

Vehicle orientation

Where is the drone facing in the NED Earth frame?

listener vehicle_attitude -n 1
2

Gimbal command

What quaternion and frame flags were sent to the gimbal?

listener gimbal_device_set_attitude -n 1
3

Gimbal result

Where does the gimbal report that it is actually pointing?

listener gimbal_device_attitude_status -n 1
Run the commands in this order and record the Euler yaw printed beside each quaternion. Also record the command flags; they determine the frame in which its yaw must be interpreted.

General listener syntax

listener <topic_name> [-i instance] [-n messages] [-r rate_hz]
CommandWhat it doesWhen to use it
listener vehicle_attitudePrints one message. One message is the default.Quick inspection.
listener vehicle_attitude -n 1Explicitly prints one message.Capturing a reproducible snapshot.
listener vehicle_attitude -n 5Prints five topic updates.Checking whether a value is changing.
listener vehicle_attitude -n 10 -r 2Prints ten messages at no more than 2 Hz.Watching motion without flooding the console.
listener <topic> -i 1 -n 1Reads instance 1 of a multi-instance topic.Distinguishing multiple sensors or publishers.
listenerPrints the command usage because no topic was supplied.Remembering the available options.

While printing messages, exit with Ctrl+C, Esc, or Q.

What to read in each message

vehicle_attitude

q rotates from vehicle FRD to Earth NED. Use the displayed yaw as the vehicle’s Earth heading. Check timestamp to make sure the sample is current.

gimbal_device_set_attitude

q is the requested orientation. Read flags before interpreting it: flag 32 means vehicle-relative yaw; flag 64 means Earth-relative yaw.

gimbal_device_attitude_status

q reports the resulting orientation. Compare its displayed yaw with your prediction. Check failure_flags before blaming frame math.

Useful flag values

4 roll lock · 8 pitch lock · 16 yaw lock · 32 vehicle-frame yaw · 64 Earth-frame yaw.

Lesson 0001 prediction checks

Command frameCorrect Earth resultWhat a frame bug looks like
Vehicle, flag 32result ≈ vehicle yaw + command yawThe gimbal correctly moves with the vehicle reference.
Earth, flag 64result ≈ command yawresult ≈ vehicle yaw + command yaw: vehicle yaw leaked into the result.
Quaternion values and Euler yaw are orientations, not angular velocities. A small difference between commanded and reported yaw can come from simulation timing, controller response, or gimbal limits; look for the full vehicle-yaw-sized error that identifies the frame bug.
