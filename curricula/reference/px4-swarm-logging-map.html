PX4 swarm logging map

Fleet collector invariants

  • Every vehicle has a unique MAV_SYS_ID; use the inbound MAVLink header source, not the payload target, as the stream identity.
  • Maintain sequence number, partial ULog message, file handle, and counters independently per vehicle.
  • Acknowledge every LOGGING_DATA_ACKED, including duplicates.
  • Use first_message_offset after loss to recover at the next ULog message boundary.
  • Write one ordinary .ulg per vehicle; keep fleet indexes and clock transforms in a separate manifest.
  • Align clocks with GPS UTC or measured time-sync offsets, never by packet arrival time alone.

Important: PX4 makes the ULog definitions/header reliable, but live data normally uses unacknowledged LOGGING_DATA. Wireless loss is expected and must remain observable.
