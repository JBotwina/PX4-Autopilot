PX4 quaternion rotations

Hamilton convention · scalar first · [w, x, y, z]

Axis-angle encoding

q = [cos(θ/2), axis · sin(θ/2)]

The vector part points along the rotation axis. The quaternion must have unit length to represent a pure rotation.

Pure yaw

q_yaw(ψ) = [cos(ψ/2), 0, 0, sin(ψ/2)]

Yaw rotates around the down/Z axis, so only w and z are nonzero.

Inverse

q⁻¹ = conjugate(q) = [w, −x, −y, −z]

This equality assumes a unit quaternion. The inverse undoes the rotation.

Composition

q_result = q₂ ⊗ q₁
first apply q₁, then apply q₂

Quaternion multiplication is not generally commutative: q₂q₁ ≠ q₁q₂.

Rotate a vector

v_rotated = q ⊗ [0,v] ⊗ q⁻¹

Represent the vector as a pure quaternion with scalar part zero.

PX4 frame conversion

q_joint = q_vehicle⁻¹ ⊗ q_target
q_vehicle ⊗ q_joint = q_target

The inverse removes the vehicle-to-earth rotation, leaving the target expressed relative to the vehicle.

Useful yaw values

YawQuaternion [w, x, y, z]Meaning
0°[1, 0, 0, 0]No rotation
+90°[0.7071, 0, 0, 0.7071]East from north
−90°[0.7071, 0, 0, −0.7071]West from north
180°[0, 0, 0, 1]South from north
360°[−1, 0, 0, 0]Same orientation as 0°

Two facts that prevent common mistakes

  • q and −q represent the same physical orientation.
  • A quaternion value is incomplete without its source and destination frames.
